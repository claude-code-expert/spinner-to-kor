# /tdd-red — 마일스톤의 실패하는 테스트 작성

Kent Beck TDD의 **RED** 단계다. **마일스톤 단위**로 진행하되, 마일스톤 내부에서는 한 테스트씩 쌓는다.

## 절차

1. `TDD-IMPLEMENTS.md`를 읽는다.
2. 대상 마일스톤을 정한다:
   - `$ARGUMENTS`가 주어지면 그 마일스톤 ID(예: `M1`).
   - 없으면 **상태가 `TODO`인 가장 위의 마일스톤** 하나.
3. 그 마일스톤의 **세부 테스트 체크리스트**를 위에서 아래로 따라가며, 각 행위를 검증하는 테스트를 작성한다.
   - 한 행위 = 한 테스트. 행위를 설명하는 의미 있는 이름을 쓴다 (예: `parses header row into menu item`).
   - 테스트 실패 메시지가 명확하고 유용하도록 단언한다.
   - 도메인 로직은 순수 함수로 호출하는 것을 전제로 import 한다. fixture(샘플 워크북 객체 등)가 필요하면 테스트 안에서 만든다.
   - 마일스톤의 체크리스트 항목을 모두 덮는 테스트를 작성한다.
4. `npm test`를 실행해 마일스톤의 새 테스트들이 **실패(RED)** 하는지 확인한다.
   - 통과해 버리는 테스트는 새 행위를 요구하지 않은 것이다 → 더 날카롭게 고친다.
   - 실패가 "단언 실패" 또는 "모듈/함수 없음" 때문인지 확인하고, 대표 실패 줄을 기록한다.
5. `TDD-IMPLEMENTS.md`에서 해당 마일스톤 상태를 `RED`로, `테스트 파일` 칸에 경로를 적는다.

## 금지

- **구현 코드를 작성하거나 수정하지 않는다.** (테스트를 통과시키려는 어떤 src 변경도 금지)
- 한 번에 **하나의 마일스톤**만. 다음 마일스톤 항목의 테스트를 미리 쓰지 않는다.
- 백로그에 없는 행위의 테스트를 추가하지 않는다.

## 끝

여기서 멈춘다. 통과시키는 일은 `/tdd-green`이 한다. 마지막에 "RED 확인됨: <마일스톤 ID> — 실패 테스트 N개, 대표 실패 줄: <quote>" 한 줄로 보고한다.
